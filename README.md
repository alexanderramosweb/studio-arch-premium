## 🎯 Decisiones Técnicas Justificadas

### ¿Por qué GSAP en vez de Framer Motion?
**Decisión:** GSAP (GreenSock Animation Platform)

**Justificación:**
- GSAP: Librería vanilla JS, no está atada a React
- Astro renderiza HTML estático, GSAP se ejecuta al cargar
- Framer Motion = dependencia de React (overkill para Astro)
- GSAP tiene mejor control sobre timing y easing en scroll triggers
- GSAP ScrollTrigger: perfecto para sync animaciones con scroll
- Resultado: 98 PageSpeed vs 85-90 con Framer

### ¿Por qué Lenis en vez de scroll smooth CSS?
**Decisión:** Lenis para scroll suave

**Justificación:**
- CSS `scroll-behavior: smooth` es básico y no sincronizable con GSAP
- Lenis intercepta scroll nativo, proporciona RAF smooth
- Permite hijacking del scroll para animaciones personalizadas
- Funciona diferente en mobile (iOS 15+ tiene scroll physics nativos)
- Impacto visible: scroll se siente "premium" vs scroll por defecto

### ¿Por qué Astro 6 sin React/Vue?
**Decisión:** Astro 6 con componentes .astro puros

**Justificación:**
- 0 JavaScript innecesario en componentes visuales
- GSAP se ejecuta en el cliente sin competencia de React hydration
- Tiempo de carga: HTML estático renderizado en servidor
- Animaciones: cargan DESPUÉS de que HTML está listo (no en paralelo)
- Resultado: FCP rápido + animaciones suaves después

### ¿Por qué Tailwind v4 específicamente?
**Decisión:** Tailwind CSS v4 (no v3)

**Justificación:**
- Tailwind v4 tiene motor CSS renovado (menos overhead)
- @apply mejorado para componentes con animaciones
- Custom properties nativas integradas
- Purging más inteligente (elimina CSS no usado de GSAP)
- Mejor soporte para variables CSS animables

### ¿Cómo sincronizar GSAP + Lenis sin conflictos?
**Decisión:** Lenis inicializa PRIMERO, GSAP se ajusta

**Justificación:**
```
1. Lenis intercepta el scroll nativo
2. Proporciona smooth scroll value en RAF
3. GSAP ScrollTrigger lee la posición de Lenis (no del scroll nativo)
4. Animaciones se lanzan según posición suave de Lenis
5. Result: scroll suave + animaciones sincronizadas
```

Si hicieras al revés, habrían conflictos de scroll.

## 🎬 Desglose de Animaciones (Por Sección)

### Hero Section
- **Parallax de imagen:** Imagen de fondo se mueve más lento que scroll (depth effect)
- **Fade in de texto:** Título y subtitle aparecen con opacity 0→1 en timeline
- **Scale entrance:** Botón CTA crece de scale 0.8 → 1.0

### Gallery Section
- **Stagger reveal:** Cada imagen entra con delay (0.1s entre ellas)
- **Hover effect:** Al pasar mouse, imagen se scale 1.05 + brightness +10%
- **Lazy load trigger:** GSAP detecta cuando imagen entra en viewport, ENTONCES carga

### Quote Section
- **Split text animation:** "Una experiencia residencial..." → cada palabra vuela de izq a derecha
- **Parallax subtle:** El quote se mueve ligeramente diferente a background
- **Fade color:** El color del texto cambia gradient al scroll (verde → azul)

### Services Section
- **Counter animation:** Número salta de 0 → valor final (ej: 0 → 50 residencias)
- **Icon entrance:** SVG de cada servicio aparece con rotate + scale
- **List stagger:** Puntos de texto aparecen en cascada

### Contact CTA
- **Pulse animation:** Botón "Contacto" tiene pulse subtle (scale 1.0 → 1.03 → 1.0)
- **Text underline:** Al hover, subrayado del botón crece de 0% → 100% width

## 🔧 Desafíos Resueltos

### Desafío 1: Animaciones sin sacrificar PageSpeed
**Problema:** GSAP + Lenis = JavaScript pesado, típicamente 80-85 PageSpeed
**Solución:**
- Minimizar GSAP: solo ScrollTrigger + Tween, no Draggable/Flip
- Code split: animaciones se cargan lazy (después de DOM ready)
- Lenis: configurar con minimal DOM thrashing (RAF optimizado)
- Debounce resize listeners (no recalcular animaciones cada 16ms)
- Resultado: 98 PageSpeed CON animaciones complejas

### Desafío 2: Sincronizar Lenis + GSAP sin conflictos
**Problema:** Scroll nativo vs Lenis smooth vs GSAP ScrollTrigger = conflicts
**Solución:**
```javascript
// Orden crítica:
// 1. Inicializar Lenis primero
const lenis = new Lenis();

// 2. Conectar Lenis a GSAP
gsap.registerPlugin(ScrollTrigger);
lenis.on('scroll', ScrollTrigger.update);

// 3. AHORA crear ScrollTriggers
gsap.to('.element', {
  scrollTrigger: {
    trigger: '.element',
    // ScrollTrigger ahora usa la posición suave de Lenis
  }
});
```
Resultado: scroll suave + animaciones pixel-perfect

### Desafío 3: Imágenes grandes sin perder rendimiento
**Problema:** Gallery con 12+ imágenes de arquitectura (cada una 3-5MB)
**Solución:**
- WebP + JPEG fallback
- Lazy load con intersection observer (GSAP lo maneja)
- srcset para diferentes tamaños de pantalla
- Compresión con Imagemin (70% reducción)
- Progressive JPG (carga borrosa → clara gradualmente)
- Resultado: Gallery carga en <2s, lazy images cargan on-demand

### Desafío 4: Tipografía custom sin aumentar LCP
**Problema:** Fuente custom elegante = extra request + font-display delay
**Solución:**
- Font-display: swap (mostrar fallback inmediatamente)
- Preload de fuente en <head> con rel="preload"
- WOFF2 (más pequeño, moderno)
- Limitar a 2 weights (regular + bold, no 4+ weights)
- Resultado: LCP no aumenta, fuente es crisp una vez lista

## 📑 Secciones Detalladas

### Navbar Sticky
- Altura: 80px (desktop), 60px (mobile)
- Background: black con opacity 0.95 (blur backdrop)
- Menú items: Proyectos, Nosotros, Contacto (smooth scroll)
- Logo: Custom wordmark (no imagen, SVG puro)

### Hero
- Imagen background: Parallax effect (moves slower than scroll)
- Headline: "Espacios diseñados para una nueva forma de vivir"
- Subheadline: Explicación breve (1-2 líneas)
- CTA: "Descubrir proyectos" (smooth scroll a projects)

### About Section
- Texto descriptivo: 3-4 párrafos máximo
- Imagen secundaria (pequeña, lado derecho)
- Copy enfatiza: filosofía de diseño, no features técnicas

### Features
- 3 cards: Wellness, Rooftop, Concierge
- Cada card: icono + título + descripción
- Animación: entrada en cascada (stagger)

### Gallery
- Grid 3 columnas (desktop), 2 (tablet), 1 (mobile)
- Imágenes: cuadradas, aspecto ratio 1:1
- Hover: zoom 1.05 + brightness +15%
- Lazy loading: imágenes se cargan al entrar en viewport

### Featured Project
- Sección grande: imagen + texto lado a lado
- Texto: Ubicación, número de residencias, descripción
- Animación: imagen entra desde izquierda, texto desde derecha

### Quote
- Fuente grande (h2)
- Animación: cada palabra entra de forma individual
- Styling: cursiva o serif elegante

### Contact CTA
- Botón grande: "Solicitar información" o "Agenda presentación"
- Debajo: teléfono + email + ubicación
- Pulse animation en hover

## 📊 Comparativa: Aura Marina vs Studio Arch

| Aspecto | Aura Marina | Studio Arch |
|---------|------------|------------|
| **Objetivo** | Captar leads | Mostrar portafolio |
| **Animaciones** | Mínimas (solo smooth scroll) | GSAP + Lenis complejas |
| **Enfoque** | Conversion-first | Experience-first |
| **PageSpeed** | 99 (sin JS pesado) | 98 (con GSAP/Lenis) |
| **FormSpree** | Sí, central | Sí, pero secundario |
| **Gallery** | 3 modelos de villa | 10+ proyectos arquitectónicos |
| **Typography** | Sans (Tailwind default) | Serif + Sans (elegante) |
| **Scroll behavior** | Smooth CSS | Lenis (hijacked) |
| **Mobile CTA** | Botón flotante + form | Botón inline + form |
| **Target** | Compradores de villas | Arquitectos, inversores |
